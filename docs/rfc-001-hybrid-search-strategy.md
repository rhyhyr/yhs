# [RFC-001] 하이브리드 검색 전략 결정

## 상태

논의 중 (GitHub Discussions에서 진행)

## 제안 배경

유학생 비자·학사 안내 시스템에서 사용자 질문에 정확한 조항을 근거로 제시하려면 단순 키워드 검색 이상의 검색 전략이 필요합니다. 현재 고려 중인 전략과 각각의 트레이드오프를 팀과 논의하기 위해 RFC를 작성합니다.

## 제안 내용

**벡터 검색 + 그래프 검색 하이브리드 방식**을 채택합니다.

- 벡터 검색: SentenceTransformers를 이용한 문맥 기반 유사도 검색
- 그래프 검색: Neo4j를 이용한 비자 자격 간 관계 탐색
- 결과 병합: 두 검색 점수를 가중 합산하여 재순위화

## 논의 포인트

1. 벡터 검색과 그래프 검색의 점수 가중치를 어떻게 설정할 것인가?
2. 그래프 검색 없이 벡터 검색 단독으로 충분하지 않은가?
3. Neo4j 대신 가벼운 그래프 라이브러리(NetworkX 등)로 대체 가능한가?
4. 응답 지연(latency) 허용 범위는 어느 정도인가?

## 기대 결과

- 팀의 피드백을 반영하여 최종 전략을 ADR-0001에 확정 기록
- 점수 가중치 기본값 결정
