# PRD: Campus Navigator RAG

## 1. 문서 정보
- 버전: v0.1 (초안)
- 작성일: 2026-04-08
- 작성 목적: 학기 프로젝트 구현을 위한 제품 요구사항 정리

## 2. 제품 개요
Campus Navigator RAG는 대학 학사 공지/규정 문서를 기반으로 학생 질문에 대해 근거 중심 답변을 제공하는 질의응답 시스템이다. 벡터 검색과 그래프 검색을 결합해 정확도와 추적 가능성을 동시에 높인다.

## 3. 목표와 비목표
### 목표
1. 학생의 학사 정보 탐색 시간 단축
2. 답변마다 신뢰 가능한 근거 제시
3. 학기 내 재현 가능한 성능 지표 확보

### 비목표
1. 법적/행정적 최종 판단 자동화
2. 모든 대학 시스템과의 실시간 연동
3. 완전한 챗봇 상용화 수준 UI 구현

## 4. 대상 사용자
1. 학부생: 수강/장학/졸업/학사일정 질의
2. 조교/학생회: 공지 안내를 위한 빠른 확인
3. 프로젝트 팀원: 데이터/모델 개선 및 운영

## 5. 사용자 시나리오
1. 학생이 "복수전공 신청 기한"을 질문한다.
2. 시스템이 관련 공지와 학칙 문서를 검색한다.
3. 답변과 함께 출처 문서명/링크/발췌 문단을 제시한다.
4. 신뢰 점수가 낮으면 "추가 확인 필요"를 안내한다.

## 6. 핵심 기능 요구사항

### FR-1 하이브리드 검색 질의응답
- 입력: 자연어 질문
- 처리: 벡터 검색 + 그래프 기반 확장 검색
- 출력: 답변 텍스트 + 근거 목록 + 신뢰 점수
- 수용 기준:
  1. 상위 근거 최소 2개를 제시할 수 있어야 함
  2. 근거 부족 시 불확실성 메시지 반환

### FR-2 문서 인덱싱 파이프라인
- 입력: PDF/웹 원문
- 처리: 정제, 청킹, 임베딩, 개체/관계 추출, 그래프 저장
- 출력: 검색 가능한 인덱스(벡터 + 그래프)
- 수용 기준:
  1. 중복 문서 재처리 방지(해시 기반)
  2. 신규/변경 문서만 증분 반영

### FR-3 품질 검증 및 회귀 테스트
- 입력: 사전 정의 질문셋
- 처리: 자동 질의 실행, 지표 수집
- 출력: 정확도/지연시간 리포트
- 수용 기준:
  1. Fast 경로 평균 5초 이내
  2. Deep 경로 평균 10초 이내
  3. 실패 케이스 로그 저장

## 7. 비기능 요구사항
1. 성능: Fast/Deep SLA 준수
2. 신뢰성: 예외 발생 시 우아한 실패(안내 메시지)
3. 추적성: 답변-근거 매핑 가능
4. 보안: API 키/.env 로컬 보관, 로그 민감정보 마스킹
5. 유지보수성: 모듈 분리, 테스트 스크립트 제공

## 8. 데이터 요구사항
1. 입력 데이터: 학사공지 PDF, 학칙, 장학 규정, FAQ
2. 메타데이터: 문서명, 발행일, URL, 카테고리
3. 저장 구조: Document-Category-Chunk 그래프 + 임베딩
4. 품질 관리: 중복/깨진 텍스트/오탐 관계 정제 규칙

## 9. 지표(KPI)
1. Top-3 근거 포함 정확도
2. 출처 제시율(답변 중 출처 포함 비율)
3. 평균 응답시간(Fast/Deep 분리)
4. 미답변/저신뢰 비율

## 10. 리스크 및 대응
1. 문서 품질 불균일
- 대응: 전처리 규칙 강화, OCR 대상 분리
2. 환각(hallucination) 응답
- 대응: 근거 없으면 답변 제한 정책 적용
3. API 비용/속도 제약
- 대응: 로컬 모델 폴백 및 캐시 전략

## 11. 릴리스 계획 (16주)
1. W1-W4: 요구사항/데이터/파이프라인 기반 구축
2. W5-W8: MVP 검색/QA/출처 제시
3. W9-W12: 라우팅/평가/자동화 고도화
4. W13-W16: 안정화/문서화/최종 발표

## 12. 오픈 이슈
1. 대상 대학 도메인 범위(단일/복수)
2. 웹 UI 구현 범위(CLI 중심 vs 최소 웹)
3. 최종 평가지표 가중치 확정
