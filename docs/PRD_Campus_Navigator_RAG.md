# PRD: Global Campus Visa Navigator RAG

## 1. 문서 정보
- 버전: v0.2 (PDF 반영 수정본)
- 작성일: 2026-04-08
- 작성 목적: 학기 프로젝트 구현을 위한 제품 요구사항 정리

## 2. 제품 개요
Global Campus Visa Navigator RAG는 외국인 유학생 대상 대학생활 안내, 비자 체류자격 안내, 출입국 법령 정보를 통합 검색해 근거 중심 답변을 제공하는 시스템이다. 벡터 검색과 그래프 검색을 결합해 정확도와 추적 가능성을 동시에 높인다.

현재 핵심 데이터 코퍼스:
1. 2024 외국인유학생 대학생활안내(국문)
2. 비자 내비게이터(국문)
3. 출입국관리법 시행령(2025.06.01 시행)

## 3. 목표와 비목표
### 목표
1. 외국인 유학생의 비자·학사 정보 탐색 시간 단축
2. 답변마다 신뢰 가능한 근거(문서/절/조항) 제시
3. 학기 내 재현 가능한 성능 지표 확보

### 비목표
1. 법적/행정적 최종 판단 자동화
2. 모든 대학 시스템과의 실시간 연동
3. 완전한 챗봇 상용화 수준 UI 구현

## 4. 대상 사용자
1. 외국인 유학생: 체류자격, 신고의무, 학사 적응 질의
2. 국제처/조교: 안내 응답 초안 확인
3. 프로젝트 팀원: 데이터/모델 개선 및 운영

## 5. 사용자 시나리오
1. 유학생이 "D-2에서 D-10으로 전환 가능한가?"를 질문한다.
2. 시스템이 비자 내비게이터와 관련 법령 조항을 우선 검색한다.
3. 답변과 함께 문서명/해당 절·조항/발췌 문단을 제시한다.
4. 근거 부족 또는 충돌 시 "추가 확인 필요"와 공식 문의 채널(예: 1345) 안내를 제공한다.

## 6. 핵심 기능 요구사항

### FR-1 하이브리드 검색 질의응답
- 입력: 자연어 질문
- 처리: 벡터 검색 + 그래프 기반 확장 검색 + 문서 타입별 가중치
- 출력: 답변 텍스트 + 근거 목록 + 신뢰 점수 + 최신성 정보
- 수용 기준:
  1. 상위 근거 최소 2개(법령 질문은 1개 이상 법령 근거 포함)
  2. 근거 부족 시 단정 답변 없이 불확실성 메시지 반환

### FR-2 문서 인덱싱 파이프라인
- 입력: PDF/웹 원문
- 처리: 정제, 청킹, 임베딩, 조항/절 메타데이터 추출, 그래프 저장
- 출력: 검색 가능한 인덱스(벡터 + 그래프)
- 수용 기준:
  1. 중복 문서 재처리 방지(해시 기반)
  2. 신규/변경 문서만 증분 반영
  3. 시행일/개정일 메타데이터를 검색 결과에 포함

### FR-3 품질 검증 및 회귀 테스트
- 입력: 사전 정의 질문셋
- 처리: 자동 질의 실행, 지표 수집
- 출력: 정확도/지연시간 리포트
- 수용 기준:
  1. Fast 경로 평균 5초 이내
  2. Deep 경로 평균 10초 이내
  3. 실패 케이스 로그 저장
  4. 근거 포함률 95% 이상

## 7. 비기능 요구사항
1. 성능: Fast/Deep SLA 준수
2. 신뢰성: 예외 발생 시 우아한 실패(안내 메시지)
3. 추적성: 답변-근거 매핑 가능
4. 보안: API 키/.env 로컬 보관, 로그 민감정보 마스킹
5. 유지보수성: 모듈 분리, 테스트 스크립트 제공
6. 최신성: 시행일/개정일 기준 최신 문서 우선
7. 안전성: 법적 판단/행정 확정 표현 금지 정책 준수

## 8. 데이터 요구사항
1. 입력 데이터: 대학생활안내 PDF, 비자 내비게이터 PDF, 출입국 법령 PDF
2. 메타데이터: 문서명, 발행일/시행일, URL, 문서타입, 조항 정보
3. 저장 구조: Document-Category-Chunk 그래프 + 임베딩
4. 품질 관리: 중복/깨진 텍스트/오탐 관계 정제 규칙

## 9. 지표(KPI)
1. Top-3 근거 포함 정확도
2. 출처 제시율(답변 중 출처 포함 비율)
3. 평균 응답시간(Fast/Deep 분리)
4. 미답변/저신뢰 비율
5. 최신 문서 참조율

## 10. 리스크 및 대응
1. 문서 품질 불균일
- 대응: 전처리 규칙 강화, OCR 대상 분리
2. 환각(hallucination) 응답
- 대응: 근거 없으면 답변 제한 정책 적용
3. API 비용/속도 제약
- 대응: 로컬 모델 폴백 및 캐시 전략
4. 법령 개정으로 인한 답변 노후화
- 대응: 개정/시행일 기반 우선순위 + 정기 재인덱싱 스케줄

## 11. 릴리스 계획 (16주)
1. W1-W4: 비자·학사·법령 코퍼스 정규화, 스키마 설계
2. W5-W8: 하이브리드 검색 MVP, 근거/조항 제시 구현
3. W9-W12: Fast/Deep 라우팅 및 회귀평가 자동화
4. W13-W16: 최신성 관리, 안정화, 최종 데모

## 12. 오픈 이슈
1. 대상 대학 도메인 범위(단일/복수)
2. 다국어 응답 범위(한/영/중) 우선순위
3. 최종 평가지표 가중치 확정
