# OSS 라이선스 비교 분석

## 비교 대상

주요 오픈소스 라이선스 3종을 비교합니다.

## 비교표

| 항목 | MIT | Apache 2.0 | GPL v3 |
|---|---|---|---|
| **사용** | 자유 | 자유 | 자유 |
| **수정** | 자유 | 자유 | 자유 (공개 의무) |
| **배포** | 자유 | 자유 | 소스 공개 의무 |
| **특허 조항** | 없음 | 명시적 특허 허가 포함 | 포함 |
| **상업적 사용** | 허용 | 허용 | 허용 (단, 소스 공개) |
| **카피레프트** | 없음 | 없음 | 강한 카피레프트 |
| **고지 의무** | 저작권·라이선스 고지 | 저작권·라이선스·변경사항 고지 | 저작권·라이선스 고지 |
| **주요 사용 프로젝트** | React, Rails | Android, Kubernetes | Linux, WordPress |

## 각 라이선스 요약

### MIT
- 가장 단순하고 허용적인 라이선스
- 저작권 고지만 유지하면 자유롭게 사용·수정·배포 가능
- 독점 소프트웨어에 포함 가능

### Apache 2.0
- MIT보다 조건이 조금 더 많지만 여전히 허용적
- 명시적 특허 허가 조항 포함 → 특허 분쟁 리스크 완화
- 변경사항 고지 의무 있음

### GPL v3
- 강한 카피레프트: 수정·배포 시 동일 라이선스로 소스 공개 필수
- 상용 독점 소프트웨어와 결합 불가
- 커뮤니티 중심 프로젝트에 적합

## 선택 기준 및 결정

이 프로젝트는 **MIT 라이선스**를 채택합니다.

| 기준 | 이유 |
|---|---|
| 학술·연구 목적 | 제약 없이 누구나 참고·활용 가능해야 함 |
| 유학생 지원 도구 | 공공성을 위해 최대한 개방적 라이선스 선택 |
| 특허 이슈 없음 | Apache 2.0의 특허 조항이 필요하지 않음 |
| 단순성 | MIT가 가장 이해하기 쉽고 관리가 간단함 |
