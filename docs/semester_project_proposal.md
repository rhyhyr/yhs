# 학기 프로젝트 제안서 (초안)

## 1. 프로젝트명

Campus Navigator RAG: 학사 공지/규정 기반 질의응답 학습 도우미

## 2. 문제정의

대학생은 수강신청, 장학금, 졸업요건, 학사일정과 같은 정보를 학교 공지/규정 PDF와 웹페이지에서 직접 찾아야 한다. 정보가 분산되어 있고 문서 표현이 복잡해, 필요한 근거를 빠르게 찾기 어렵다. 기존 키워드 검색은 문맥 이해가 약해 정확한 답변과 출처 제시에 한계가 있다.

이 프로젝트는 Graph + Vector Hybrid RAG를 이용해 학사 문서를 구조화하고, 질문 의도에 맞는 근거를 찾아 출처와 함께 답변해 정보 탐색 시간을 줄이는 것을 목표로 한다.

## 3. 핵심 기능 (3개)

### 기능 1) 하이브리드 검색 질의응답
- 벡터 검색(의미 기반) + 그래프 검색(관계 기반) 결합
- 질문 유형에 따라 Fast Path / Deep Path 자동 분기
- 답변에 신뢰 점수와 근거 문서 스니펫 포함

### 기능 2) 학사 문서 자동 수집/인덱싱 파이프라인
- PDF/웹 공지 로딩, 정제, 청킹, 임베딩 생성
- 카테고리/개체/관계 추출 후 그래프 DB 저장
- 증분 업데이트(변경 문서만 재처리)

### 기능 3) 출처 중심 응답 및 품질 검증
- 답변마다 출처 링크, 문서명, 문단 인용 제공
- 회귀 질문셋으로 정확도/지연시간(SLA) 자동 점검
- 미답변/저신뢰 질문은 보완 후보 데이터로 수집

## 4. 기술 스택

- Language: Python 3.11+
- Backend/Orchestration: Python, CLI Runner
- LLM: Gemini/OpenAI/Ollama (환경별 선택)
- Retrieval: SentenceTransformers, Hybrid Retrieval (Vector + Graph)
- Graph DB: Neo4j
- Data Processing: pdfplumber, custom chunker/cleaner/extractor
- Evaluation: 회귀 테스트 스크립트, latency 로그
- DevOps/Collab: Git, GitHub, GitHub Issues/Projects, Actions(선택)

## 5. 16주 마일스톤 (1학기 기준)

### 1-2주: 문제 구체화 및 요구사항 정의
- 대상 사용자(학생/조교) 인터뷰 또는 시나리오 수집
- 핵심 질문셋 50개 정의
- 성공 지표(정확도, 응답속도, 출처충실도) 확정

### 3-4주: 데이터 소스 확보 및 전처리 설계
- 학사 공지/PDF/규정 데이터 수집
- 문서 스키마/메타데이터 설계
- 청킹 전략 비교 실험

### 5-6주: 인덱싱 파이프라인 구축
- 로더-클리너-청커-임베더 파이프라인 구현
- Neo4j 저장 모델 확정
- 초기 데이터셋 인덱싱 완료

### 7-8주: 하이브리드 검색 MVP
- 벡터 검색 + 그래프 검색 결합
- 기본 질의응답 루프 및 출처 표기 구현
- 중간 점검 데모

### 9-10주: Fast/Deep 라우팅 고도화
- 질문 복잡도 기반 라우팅 규칙 개선
- 근거 부족 시 후속 검색/확장 검색 적용
- 응답 형식 표준화

### 11-12주: 품질평가/회귀테스트 자동화
- 테스트 질문셋 기반 정확도 측정
- SLA(예: Fast 5초, Deep 10초) 모니터링
- 실패 케이스 분석 및 데이터 보강

### 13-14주: 사용자 경험 개선
- FAQ 우선 응답/정책 기반 예외 처리
- 로그/모니터링 정리, 오류 대응 강화
- 발표용 시나리오/데모 데이터 준비

### 15주: 통합 안정화
- 성능/정확도 튜닝
- 문서화(설치, 실행, 한계, 향후 계획)

### 16주: 최종 발표 및 결과 보고
- 최종 시연
- 성과 지표/회고
- 2학기 확장 계획 제안
