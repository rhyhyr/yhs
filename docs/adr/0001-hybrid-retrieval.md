# ADR-0001: 하이브리드 검색 전략 채택 (Vector + Graph)

## 상태

Accepted

## 날짜

2026-04-30

## 배경 및 문제

외국인 유학생 비자·학사 안내 시스템에서 사용자 질문에 정확한 조항을 찾아 제시해야 한다. 단순 키워드 검색은 문맥을 이해하지 못하고, 벡터 검색만으로는 자격 요건·전환 관계 등 구조적 정보를 표현하기 어렵다.

## 결정

벡터 검색(SentenceTransformers 임베딩)과 그래프 검색(Neo4j)을 결합한 하이브리드 검색을 채택한다.

- **벡터 검색**: 문맥 의미 기반 유사 청크 탐색
- **그래프 검색**: 비자 자격 간 관계(전환 가능 여부, 의무 조건 등) 탐색
- **결과 병합**: 두 검색 결과를 점수 기반으로 재순위화하여 최종 근거 제시

## 대안 검토

| 대안 | 장점 | 단점 | 선택 여부 |
|---|---|---|---|
| 키워드 검색(BM25) | 구현 단순, 속도 빠름 | 문맥 이해 불가, 동의어 처리 어려움 | ❌ |
| 벡터 검색 단독 | 문맥 의미 파악 가능 | 구조적 관계(자격 전환 등) 표현 한계 | ❌ |
| 그래프 검색 단독 | 관계 탐색에 강함 | 자연어 질문 매핑 어려움 | ❌ |
| **하이브리드 (Vector + Graph)** | 문맥 + 구조 모두 커버 | 구현 복잡도 증가 | ✅ |

## 결과 및 영향

- 답변 품질 향상: 단순 유사도 기반이 아닌 관계 기반 근거 제시 가능
- 구현 복잡도 증가: Neo4j 연동 및 파이프라인 관리 필요
- 후속 작업: 청킹 전략, 조항 단위 메타데이터 스키마 설계 필요
